CPL Software

Web application overhaul with added functionality

Overview

Real Advantage Title Insurance Company (RATIC) is a child company to Orange Coast Title. The web application they had before me was old not only needed a design refresh but also needed the added functionality of being able to work with companies from outside the "family of companies".

The goal of this project was to identify any usability issues with the initial release as well as add the ability to handle outside users to create and find Cover Protection Letters (CPLs).

My role

UI/UX and Front-End Development
User Research, Interaction, Visual Design, Prototyping & Testing, and Front-end development

Background

Before, working with internal clients was very straight-forward but now that the company was working towards opening up the service to the third party clients, we needed to consider all the different processes that potential customers already implement in their everyday workflow.

We needed to do this in a way where the user-experience did not feel confusing or over-crowded or else we would jeopardize the possibility of the user getting frustrated and going back to something they were already familiar with. Another problem we faced was a way for Administrators to oversee new “Entities” being added from both internal companies and new third-party companies.

Users of the system had very little complaints with the overall functionality of the sight but did complain about some usability issues, such as not being able to find previously made CPLs, that we decided to address.

Pain points and new functionality

I conducted research interviews with some of our primary users to uncover any pain points they experienced with the active version.

Top pain points identified:

  • Not being able to find previously made CPLs
  • Not being able to read the forms fields that were required
  • There was too many steps to get to creating a CPL (96% of users were only there to create)
  • User's had to enter the same information over and over again (i.e. name, company, company address, etc)
  • It was very hard to create these reports in a mobile view as the website was not responsive.

After discussion with stakeholders, the acceptance criteria for the new version of the site would need to cover.

New functionality would include:

  • Ability to add new outside entities to the system.
  • Each entity would contain an array Issuing Agent Offices, and each Issuing Agent Office contains the users associated with that office
  • Tiered user ranking where each user would be either a User, Admin, or Super Admin.
    • Users have no control over assigning offices, what office they or others are associated wit.
    • Admin are able to Manage all of the Issuing Agent Offices and their users within their entity.
    • Super Admin are able to control all Entities, Issuing Agent Offices, and Users. Super admin was limited mainly to the Development team and stakeholders of the company.
  • Reference Libraries containing title insurance information, rate estimate calculators, and a way to manage the content of their account, and if they are an admin member, the site.

Final solutions

RATIC Homepage

We ensured the login section on the homepage is highly visible. During user interviews, many new users expressed difficulty in locating the login option so we fixed it.

Splash screen post login

For efficiency, we made creating a CPL the first action users see upon logging in, as 56% of users primarily logged in just to create one and log out.

Edit Entities

As a Super Admin, you have the ability to edit both parent entities and all associated child entities in one streamlined, easy-to-follow workflow.

CPL creation on Mobile

We enhanced the CPL creation workflow by storing user information, eliminating the need to repeatedly enter the same details.
